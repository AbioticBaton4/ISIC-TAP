# 1.1 Creación de Interfaz Gráfica para Usuarios

La creación de una GUI (Graphical User Interface) en Java implica entender la transición tecnológica y la jerarquía de objetos que componen la ventana. Java utiliza un modelo basado en componentes donde cada elemento visual es un objeto que reside dentro de otro.

## Evolución Tecnológica: AWT vs. Swing

Para comprender la creación de interfaces actuales, es fundamental distinguir las dos bibliotecas base de Java:

* **AWT (Abstract Window Toolkit):**
    * Es la librería original de Java.
    * Utiliza componentes "pesados" (heavyweight), lo que significa que dependen directamente de las bibliotecas gráficas del sistema operativo subyacente (Windows, Linux, macOS).
    * Limitación: La apariencia cambia según el sistema operativo y tiene menos componentes disponibles.

* **Swing:**
    * Es una extensión de AWT y parte de las Java Foundation Classes (JFC).
    * Utiliza componentes "ligeros" (lightweight) escritos casi totalmente en Java.
    * Ventaja: Ofrece una apariencia consistente (Look and Feel) en cualquier plataforma y permite componentes más complejos como tablas y árboles.

---

## ¿Qué es un Contenedor?

Un contenedor es un componente especial cuya función principal es alojar y organizar otros componentes. En el modelo de Swing, todo componente debe estar contenido en una jerarquía que termine en un contenedor de nivel superior.

### Funciones principales:
1. **Alojamiento:** Provee el espacio físico y el contexto de dibujo para los elementos hijos.
2. **Gestión de Diseño (Layout):** Implementa un Layout Manager que decide el tamaño y la ubicación de los componentes automáticamente.
3. **Agrupación Lógica:** Permite gestionar conjuntos de elementos como una unidad (por ejemplo, ocultar un panel entero con todos sus botones).

---

## Estructura y Jerarquía de Contenedores

La arquitectura de una interfaz en Java se organiza de forma piramidal:

### 1. Top-Level Containers (Nivel Superior)
Son las ventanas que interactúan directamente con el sistema de ventanas del SO. No pueden estar dentro de otros contenedores.
* **`JFrame`:** La ventana principal con botones de maximizar, minimizar y cerrar.
* **`JDialog`:** Ventanas secundarias o emergentes para mensajes o captura de datos específicos.


### 2. Intermediate Containers (Intermedios)
Su propósito es organizar la distribución interna de la ventana.
* **`JPanel`:** El contenedor más versátil; funciona como una caja invisible para agrupar componentes.
* **`JScrollPane`:** Provee barras de desplazamiento para componentes que exceden el tamaño visible.
* **`JTabbedPane`:** Permite organizar múltiples paneles mediante pestañas.

### 3. Atomic Components (Atómicos)
Son los elementos finales de interacción que no pueden contener a otros.
* **Entrada y Acción:** `JButton`, `JTextField`, `JCheckBox`, `JRadioButton`.
* **Información:** `JLabel`, `JProgressBar`.

---

## El Content Pane (Panel de Contenido)

Es un concepto crítico en la creación de interfaces con Swing. Un `JFrame` no permite añadir componentes directamente a su estructura base; estos deben añadirse al **Content Pane**.

* **Acceso:** Se obtiene mediante el método `getContentPane()`.
* **Operación:** Por defecto, es donde se establecen los Layout Managers y se agregan los botones, etiquetas y paneles intermedios.
