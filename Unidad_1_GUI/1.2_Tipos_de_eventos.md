# 1.2 Tipos de Eventos

En el mundo de la programación visual, un evento es una señal enviada por el sistema que indica que "algo ha sucedido". Generalmente, este algo es una interacción del usuario con un componente.

## El Modelo de Delegación de Eventos

Java utiliza este modelo para separar la lógica de la interfaz de la lógica de negocio. Se basa en tres pilares fundamentales:

1. **Fuente (Source):** El componente que genera el evento (ej. un `JButton`). Es el objeto que cambia de estado.
2. **Evento (Event):** El objeto que describe el cambio de estado. Contiene información útil como la posición del ratón o la tecla presionada (ej. `ActionEvent`).
3. **Oyente (Listener):** El objeto que espera el evento y lo procesa. Para ser un oyente, una clase debe implementar la interfaz correspondiente.

---

## Clasificación Detallada de Eventos

A continuación se desglosan las categorías más comunes utilizadas en el desarrollo de aplicaciones de escritorio:

### A. Action Events (Eventos de Acción)
Es el evento de nivel más alto. No se enfoca en "cómo" interactuó el usuario (si fue con el mouse o teclado), sino en que se ejecutó la acción principal del componente.
* **Interfaz:** `ActionListener`
* **Método principal:** `actionPerformed(ActionEvent e)`
* **Uso común:** Clic en botones (`JButton`), selección en un menú (`JMenuItem`) o presionar Enter en un `JTextField`.

### B. Mouse Events (Eventos de Ratón)
Permite rastrear la interacción física detallada con el puntero. Se divide en dos interfaces según la complejidad requerida:
* **Interfaz:** `MouseListener`
  * Métodos: `mouseClicked`, `mouseEntered`, `mouseExited`, `mousePressed`, `mouseReleased`.
* **Interfaz de Movimiento:** `MouseMotionListener`
  * Métodos: `mouseDragged`, `mouseMoved`.

### C. Key Events (Eventos de Teclado)
Detecta la interacción con las teclas físicas cuando un componente tiene el foco.
* **Interfaz:** `KeyListener`
* **Métodos:**
    * `keyPressed`: Se dispara al bajar la tecla (útil para teclas especiales como Flechas o F1).
    * `keyReleased`: Se dispara al soltar la tecla.
    * `keyTyped`: Se dispara cuando se genera un carácter Unicode (letras, números, símbolos).

### D. Window Events (Eventos de Ventana)
Controla el ciclo de vida del `JFrame`. Es esencial para gestionar qué sucede antes de que una aplicación se cierre.
* **Interfaz:** `WindowListener`
* **Eventos clave:** `windowOpened`, `windowClosing` (cuando el usuario pulsa la X), `windowIconified` (minimizar), `windowActivated`.

## Tabla Comparativa: Paquetes y Clases

| Tipo de Evento | Clase de Evento | Interfaz de Escucha | Paquete |
| :--- | :--- | :--- | :--- |
| Acción | `ActionEvent` | `ActionListener` | `java.awt.event` |
| Ratón | `MouseEvent` | `MouseListener` | `java.awt.event` |
| Teclado | `KeyEvent` | `KeyListener` | `java.awt.event` |
| Ventana | `WindowEvent` | `WindowListener` | `java.awt.event` |
| Cambio de Ítem | `ItemEvent` | `ItemListener` | `java.awt.event` |

> **Nota Importante:** Todos los eventos en Swing son objetos que heredan de `java.util.EventObject`. Esto permite acceder a métodos como `getSource()`, el cual devuelve una referencia al componente que originó el evento.
